2. The Problem We Are Solving

Most enterprise BI platforms today suffer from three challenges:

Too many dashboards Organizations maintain dozens of dashboards across departments. Business users waste time searching for the right one.
Numbers without context A dashboard shows "Total Orders: 7,462,065" but doesn't explain what's included (line items or distinct transactions?), what's excluded, or how it was calculated.
Opaque data pipelines When a number looks wrong, tracing it from dashboard → data warehouse → ETL → raw source requires tribal knowledge that lives in people's heads, not in the tool.

3. How the GenBI Solution Addresses These Issues

AI-Powered Natural Language Q&A

Instead of navigating multiple dashboards, users simply ask a question in plain English. The chatbot:

  1. Retrieves context from a Bedrock Knowledge Base (schema definitions, validated SQL patterns, business glossary) using RAG (Retrieval-Augmented Generation)
  2. Generates SQL using Amazon Bedrock LLM — producing syntactically correct Redshift queries with proper JOINs and aggregations
  3. Executes the query against Amazon Redshift Serverless and returns tabular results
  4. Recommends the right dashboard for deeper visual exploration

End-to-End Data Lineage — From Source to Metric

For every answer, the chatbot traces the complete data pipeline:

Source System: POS terminals (store registers) →
S3 Raw Landing: s3://.../pos/transactions/ (monthly CSV) + s3://.../pos/line_items/ →
Glue ETL (load_fact_sales): JOIN transactions + line_items, calculate gross_profit →
Redshift genbi_mart.fact_sales: 17.5M rows, grain = one row per line item →
Dashboard Aggregation: COUNT(DISTINCT transaction_id) → order count

This gives business users complete transparency into every number they see — no more tribal knowledge required.

Topic Guardrails

The chatbot is restricted to restaurant operations topics only (sales, revenue, inventory, labor, customer satisfaction, financial P&L). Off-topic questions receive a polite redirect, ensuring the system stays focused and secure.

Model-Agnostic & Tool-Agnostic Design

The solution is built on Amazon Bedrock, which supports multiple LLM providers (Anthropic Claude, Amazon Titan, Meta Llama, Mistral, etc.). The LLM can be swapped by changing a single configuration value. Similarly, the knowledge base approach works with any ETL tool (Informatica, dbt), any data warehouse (Snowflake, BigQuery), and any BI tool (Power BI, Tableau).
