01 What is Kiro and Why It Matters

Kiro is an AI-powered development assistant that operates directly in your terminal. Unlike traditional chatbots, Kiro has deep integration with your local environment — it can read files, execute commands, interact with AWS services, analyze images, and write production-ready code.

In this project, Kiro demonstrated its ability to:

  • Understand business requirements from a screenshot — Kiro analyzed a dashboard image and extracted chart types, KPI values, data relationships, and layout structure
  • Design a normalized database schema — Created dimension and fact tables following star-schema best practices for analytics
  • Generate realistic synthetic data — Produced 6,569 orders and 14,646 line items with realistic distribution patterns (lunch/dinner peaks, weekend spikes, seasonal trends)
  • Provision cloud infrastructure — Set up Redshift Serverless schema, managed IAM permissions, and loaded data via the AWS Data API
  • Build a production-quality dashboard — Created an interactive HTML dashboard with 6 chart types, KPI cards, and hour-based filtering

Key Value: The entire process — from screenshot analysis to working dashboard with live Redshift data — was completed in a single Kiro conversation, demonstrating how AI-assisted development can compress weeks of work into hours.

SQL Setup

-- Run: sql/01_schema_and_dimensions.sql
CREATE SCHEMA IF NOT EXISTS abc_hk;

CREATE TABLE abc_hk.categories (
  category_id   INT PRIMARY KEY,
  category_name VARCHAR(50) NOT NULL
);

CREATE TABLE abc_hk.menu_items (
  item_id     INT PRIMARY KEY,
  item_name   VARCHAR(100) NOT NULL,
  category_id INT NOT NULL,
  unit_price  DECIMAL(10,2) NOT NULL
);

CREATE TABLE abc_hk.orders (
  order_id          INT PRIMARY KEY,
  order_date        DATE NOT NULL,
  order_hour        INT NOT NULL,
  order_day_of_week VARCHAR(10) NOT NULL
);

CREATE TABLE abc_hk.order_items (
  order_item_id INT PRIMARY KEY,
  order_id      INT NOT NULL,
  item_id       INT NOT NULL,
  quantity      INT NOT NULL,
  line_total    DECIMAL(10,2) NOT NULL
);

06 Step 4 — Data Loading into Redshift

Kiro used the AWS Redshift Data API to load data securely — no direct database credentials needed.

Security Approach

  1. Managed admin password — Stored in AWS Secrets Manager, never exposed in code
  2. IAM authentication — Redshift Data API uses IAM roles, not database passwords
  3. Least privilege — IAM user granted only CREATE on the specific database
  4. Dedicated schema — abc_hk schema isolates dashboard data from other workloads

Loading Commands

# 1. Grant permissions (one-time, via admin)
aws redshift-data execute-statement \
  --database dev \
  --workgroup-name demo-sales-related \
  --secret-arn "arn:aws:secretsmanager:us-east-1:ACCOUNT:secret:redshift!..." \
  --sql "GRANT CREATE ON DATABASE dev TO \"IAM:username\";" \
  --region us-east-1

# 2. Create schema and tables
aws redshift-data execute-statement \
  --database dev \
  --workgroup-name demo-sales-related \
  --sql "$(cat sql/01_schema_and_dimensions.sql)" \
  --region us-east-1

# 3. Load order data (batch inserts)
aws redshift-data execute-statement \
  --database dev \
  --workgroup-name demo-sales-related \
  --sql "$(cat sql/02_orders_data.sql)" \
  --region us-east-1

08 Step 6 — Interactive Filtering

The hour filter dynamically updates all 6 charts and all KPI cards simultaneously. Kiro queried Redshift for per-hour breakdowns across every dimension, then embedded the aggregated data in the dashboard.

How It Works

  1. User clicks an hour pill (e.g., "12") or "All"
  2. getData(hour) aggregates the pre-computed hourly data for the selected hour(s)
  3. All 6 Chart.js instances are destroyed and recreated with filtered data
  4. KPI cards recalculate totals, averages, and counts
// Filtering logic (simplified)
function getData(hour) {
  const hrs = hour === 'all' ? ALL_HOURS : [parseInt(hour)];
  const catData = mergeObjects(hrs.map(h => hourlyRevByCat[h]));
  const kpis = sumAcrossHours(hrs);
  return { catData, kpis, ... };
}

09 Best Practices for Working with Kiro

1. Provide Visual References

Screenshots are worth a thousand words. Kiro can analyze images to extract layout, chart types, color schemes, and data patterns. Always provide a reference image when building dashboards.

2. Describe Business Context

Don't just say "build a dashboard." Describe the KPIs, the audience, and the business questions. For example: "Highlight peak lunch hours to optimize staffing" gives Kiro context to weight data generation appropriately.

3. Iterate in Phases

Break complex projects into phases: schema → data → dashboard → interactivity. This lets you validate each step before moving on.

4. Let Kiro Handle Infrastructure

Kiro can directly interact with AWS services — creating schemas, managing permissions, loading data. This eliminates context-switching between your IDE and the AWS console.

5. Review Security at Each Step

Ask Kiro about security implications. In this project, Kiro used Secrets Manager for admin credentials, IAM authentication for the Data API, and dedicated schemas for isolation.

6. Request Documentation

Ask Kiro to package deliverables with documentation. This ensures the solution is reproducible and transferable to other teams.

11 Deployment Guide

Option A: S3 + CloudFront (Recommended)

Cheapest and most scalable. Ideal for static dashboards with embedded data.

# 1. Create S3 bucket
aws s3 mb s3://abc-hk-dashboard-prod --region us-east-1

# 2. Upload dashboard files
aws s3 sync ./abc-hk-dashboard s3://abc-hk-dashboard-prod/ \
  --exclude "*.py" --exclude "sql/*" --exclude "*.md"

# 3. Create CloudFront distribution
aws cloudfront create-distribution \
  --origin-domain-name abc-hk-dashboard-prod.s3.amazonaws.com \
  --default-root-object index.html

Option B: Amplify Hosting

One-click deployment with CI/CD built in.

# Connect to Git repo and deploy
aws amplify create-app --name abc-hk-dashboard
aws amplify create-branch --app-id APP_ID --branch-name main

Option C: Live Data with Lambda + API Gateway

For real-time dashboards that query Redshift on each page load.

User Browser → CloudFront → API Gateway → Lambda → Redshift Data API → Redshift
